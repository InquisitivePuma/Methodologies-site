Development Methodologies

Waterfall

The waterfall methodology consists of six stages:

Determining requirements, Analysis, Design, Coding,
Testing and maintainence.

Determining requirements involves market research,
surveys, and speaking to clients in order to find
out what exactly you need to develop. Then, you
analyze - often using the TELOS (Technology, Econoics,
Legal, Operative, timeScale) method - in order to determine
the feasibility of the project. Next, design such as making
flowcharts, heirarchy digrams and structure diagrams. Doing
this allows the coding section to be organised properly.

The coding section is where the product is actually
produced. Documentation here could include comments in
the code. After coding comes testing, and the fact that
this doesn't come until so late in the project is a
weakness of this methodology; flaws at thsi stage can send
the project all the way back to the design stage! However, if
the project makes it past this stage all that is left is
maintainence; extending the project's life cycle by adapting
it to changing enviroments by correcting, perfecting and adapting.
